## 第零步：段永平式灵魂拷问

在讨论怎么赚钱之前，先想清楚三个问题：

### 你到底在卖什么？

不是卖工具（prompt可以抄），不是卖报告（AI都能写），卖的是 **"一个连续两年跑赢指数40-50个百分点的人，把他的投资决策系统产品化"**。核心资产是 **判断力 + 验证过的方法论 + 真金白银的track record**。

### 护城河在哪？

- 开源的skill文件谁都能复制，**没有护城河**
- 四大师框架是公开知识，**没有护城河**
- 但"AI算法专家 + 价值投资实盘连续跑赢"这个交叉身份，**全中国找不出几个人**——这才是护城河

### 你愿意做十年吗？

如果只是想变现赚一笔，做个课程卖掉就行。但如果真的想做一件事，答案决定了商业模式的选择。

## 一、商业模式分析：四条路

| 路径 | 模式 | 年收入天花板 | 启动成本 | 风险 |
|------|------|------------|---------|------|
| A. 投研内容订阅 | 付费研报/newsletter | 50-200万 | 极低 | 合规风险 |
| B. SaaS工具 | 把skill做成Web产品收月费 | 500-2000万 | 高（全职+团队） | 技术+获客 |
| C. 教育/课程 | 教别人用AI做投研 | 100-500万 | 低 | 一次性、不复利 |
| D. 私募基金 | 拿真钱管理 | 无上限 | 极高（牌照+合规） | 监管+业绩回撤 |

**判断：A先行，B中期，D是终局。C可以穿插但不应该是主线。**

---

## 二、三阶段演进路径

### 第一阶段：内容变现（0-12个月，在职可做）

**核心逻辑**：先用内容建立个人品牌和信任资产，不需要辞职。

#### 1. 付费投研周报/月报

- **平台**：小报童、知识星球、或自建newsletter（Substack/竹白）
- **内容**：每月2-3篇深度研报（用自己的框架生成，人工审校把关）+ 实盘持仓更新
- **定价**：199-399元/年（低门槛圈用户）或99元/月（高客单价筛选严肃投资者）
- **差异化**：不是又一个股评号。卖点是"AI算法专家的投资系统"——既展示方法论透明度，又展示实盘结果

#### 2. 雪球/即刻/公众号建立阵地

- 免费内容引流：每周分享一个"AI投研的认知差"
- 例如："我用4个AI Agent同时分析腾讯，它们吵起来了——这就是好分析的标志"
- 实盘收益截图是最强信任背书，持续更新

#### 3. 开源项目作为漏斗入口

- ai-berkshire 保持开源免费，但README引流到付费内容
- GitHub star → 关注公众号/newsletter → 付费订阅

#### 收入预估

500-2000个付费用户 × 199-399元/年 = **10-80万/年**

**关键指标**：前6个月到500付费用户 = 验证成功

---

### 第二阶段：产品化（12-24个月，需要判断是否全职）

**核心逻辑**：把"用Claude Code跑skill"变成"用户在网页/App上一键生成研报"。

#### 产品形态

- Web应用，用户输入公司名 → 自动生成四大师分析报告
- 免费版：每月3份基础报告
- Pro版：不限量 + 多Agent深度报告 + 实时异动归因 + 组合管理
- 定价：Pro 99-199元/月

#### 技术架构

- 前端：Next.js / React
- 后端：Python，调用Claude API
- 数据层：接入Yahoo Finance / 东方财富 / 富途API
- 核心IP：prompt工程 + 多Agent编排逻辑 + 金融严谨性校验

#### 护城河构建

- **用户数据飞轮**：用户的研究历史、持仓跟踪、论文验证 → 个性化推荐 → 粘性
- **社区效应**：用户分享和对比研报 → 内容网络效应
- **持续迭代的方法论**：实盘持续跑赢 → 框架可信度持续增强

#### 收入预估

5000-20000付费用户 × 1200-2400元/年 = **600-4800万/年**

---

### 第三阶段：资管（3-5年，如果track record持续验证）

**核心逻辑**：如果连续5年跑赢指数，最大的商业价值不是卖工具，而是直接管钱。

- 发起私募基金，AI辅助投研+人工决策
- 管理费1% + 业绩报酬20%（行业标准）
- 管理1亿 = 年管理费100万 + 超额收益的20%
- 管理10亿 = 另一个量级的生意

**前提条件**：需要私募牌照、合规团队、3年以上可验证的业绩记录。

## 三、核心风险与对策

| 风险 | 严重度 | 对策 |
|------|-------|------|
| **合规风险**：中国对投资建议类内容监管严格，无牌照不能荐股 | 高 | 定位为"投资研究工具/教育"而非"投资建议"；报告标注免责声明；不给具体买卖建议，只呈现分析框架 |
| **业绩回撤**：连续跑赢不可能永远持续，一次大回撤会严重伤害信誉 | 高 | 从一开始就管理预期——强调"框架和纪律"而非"收益率"；公开分享亏损案例建立诚实人设 |
| **Claude API依赖**：产品完全依赖Anthropic的API | 中 | 架构上做模型抽象层，支持多模型后端；短期内Claude是最优选 |
| **竞争**：大平台做类似功能（雪球AI、富途AI研报） | 中 | 它们做的是"通用AI分析"，本项目做的是"经过实盘验证的特定方法论"，定位不同 |
| **在职冲突**：如果去千问，公开做AI投资产品可能有利益冲突 | 中 | 第一阶段以个人内容创作为主，不涉及产品竞争；产品化阶段需要评估 |

## 四、段永平视角的生意评估

用项目自己的框架来评估"ai-berkshire作为一门生意"：

| 维度 | 评分 | 判断 |
|------|------|------|
| 好生意？ | ★★★★☆ | 内容/SaaS都是轻资产高毛利，投入产出比好 |
| 护城河？ | ★★☆☆☆ | 目前很浅——开源prompt可复制，需要靠品牌和track record建立 |
| 管理层？ | ★★★★☆ | 技术能力强、投资有实证、但缺乏从0到1做产品/商业的经验 |
| 确定性？ | ★★★☆☆ | 取决于业绩能否持续，以及是否愿意持续投入 |
| 安全边际？ | ★★★★★ | 第一阶段几乎零成本启动，在职就能做，downside极小 |
